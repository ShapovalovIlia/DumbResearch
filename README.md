# Workflow
## Plan
### 1. **Выбор инструментов:**
   - Выберите и используйте существующие LLMAs (Large Language Models for Activities), которые подходят для поддержки ваших академических и неакадемических деятельностей, а также выберите ERT (Emotion Recognition Tool), который может автоматически определять эмоции и отображать результаты. Эмоциональный анализ может быть выполнен в моно-модальном (только голос) или мультимодальном (голос + выражения лица) режиме. Мультимодальный режим может предоставить более полные данные и лучшие инсайты по исследуемым темам.
   - Используйте тот же ERT для всех деятельностей, основанных на LLMAs. Примеры LLMAs и ERTs приведены в Приложении A (Ultra), также можно исследовать другие варианты самостоятельно.

### 2. **Взаимодействие с пользователем:**
   - Выполните метод "мысленного озвучивания" (think-aloud), вербализируя свои мыслительные процессы при взаимодействии с LLMAs, и логируйте свои действия.
   - Интегрируйте ERT для захвата эмоциональных откликов во время взаимодействия.
   - После каждого взаимодействия заполняйте анкету о доверии.
     - **Неделя 1 (4-8 ноября 2024):** Выполните взаимодействие с ERT и выбранными LLMAs для 1 академической и 1 неакадемической деятельности. Минимальное время взаимодействия — 5 минут для каждой активности. Разработайте шаблон отчета для представления сырых данных (см. Раздел 3.A: Итоговое задание, Часть 1).
     - **Неделя 2-3 (2-13 декабря 2024):** Выполните взаимодействие с ERT и LLMAs для 3 различных академических и 3 неакадемических деятельностей, каждая продолжительностью от 5 до 15 минут. Используйте доработанный шаблон отчета для представления сырых данных (см. Раздел 3.B: Итоговое задание, Часть 2).

### 3. **Цель:**
   Оценить как когнитивные (мысли и действия), так и эмоциональные данные для оценки их отдельного и комбинированного влияния на улучшение принятия решений LLMA и доверия пользователей.

## **B. Реализация**

### 1. **Сбор данных по методу "мысленного озвучивания":**
   - Используйте LLMAs для выполнения деятельности, озвучивая ваш мыслительный процесс.
   - **API для преобразования речи в текст:** Используйте любое средство преобразования речи в текст (например, Google Speech API или библиотеку Python SpeechRecognition) для преобразования вербализованных мыслей в текст.

### 2. **Сбор данных о эмоциях:**
   - В моно-модальном режиме интегрируйте инструмент распознавания эмоций по голосу.
   - В мультимодальном режиме записывайте ваше лицо с помощью веб-камеры.
   - ERT анализирует аудио/видеоданные и присваивает метки эмоций в реальном времени. Эмоциональные данные (например, временные метки с эмоциональными состояниями) регистрируются вместе с данными о мыслях и действиях. Альтернативно, вы можете записать взаимодействие и загрузить его в ERT для анализа после взаимодействия. Обеспечьте синхронизацию всех данных.

**Эмоции для захвата:**
   - **Интерес/Скука:** Это критично для понимания вовлеченности во время взаимодействия, как обсуждается в подходе "Models of human Nature of Interestingness" (OMNI).
   - **Другие эмоции:** Эмоции, возвращаемые выбранным ERT, которые вы считаете актуальными.

3. **Сбор данных о действиях:**
   - **Данные о действиях:** Записывайте действия при взаимодействии с LLMAs (например, вопросы, которые вводятся в чат-бот).
   - **Логирование данных:** Данные о действиях могут быть записаны вручную или с использованием простых скриптов для логирования событий (например, когда вводится сообщение в чат-бот, оно может сохраняться в лог-файл).
   - **Синхронизация данных:** Убедитесь, что все данные о мыслях, действиях и эмоциях записываются с временными метками для дальнейшего анализа.

**C. Анализ**

Предполагается, что вы выполните анализ по трем аспектам, перечисленным ниже. Для каждого аспекта приведены идеи и примеры. Вы можете выбрать, какие из них наиболее актуальны для ваших данных:

1. **Анализ эмоциональной траектории:**
   - **Визуализация:** Проанализируйте, как эмоциональное состояние пользователя изменяется в ходе взаимодействия. Это можно сделать с помощью графиков или визуальных временных шкал, отображающих эмоциональные отклики, особенно интерес против скуки на различных этапах задачи. Примеры инструментов: Python (Matplotlib или Seaborn) или Excel.
   - **Интерпретация:**
     - Проанализируйте, как эмоциональная траектория отражает опыт пользователя. Например, падение интереса может свидетельствовать о том, что LLMA не смогло эффективно вовлечь пользователя в какой-то момент.
     - Проанализируйте, как эмоции могут совпадать или расходиться с мыслительными процессами. Например, пользователь может выражать разочарование словами (мысли), но ERT может зафиксировать рост интереса из-за вызова, предложенного LLMA.

### 2. **Анализ данных о мыслях, действиях и эмоциях:**
   - **Анализ данных о мыслях:** Изучите данные о мыслях, чтобы выявить закономерности в принятии решений. Проанализируйте, как ваш мыслительный процесс направлял ваши действия и какие рассуждения приводили к конкретным выборам.
   - **С мыслительными данными и без них:** Представьте, как бы LLMA работало, если бы у него были только данные о действиях (например, ввод пользователя) или оба типа данных — действия и мысли (например, рассуждения пользователя).
   - **Анализ следующих сценариев:**
     - **Несоответствие мыслей и действий:** Выявите, где мысли и действия не совпадают, подчеркивая области, где данные о мыслях могли бы предотвратить ошибочное решение.
     - **Предсказание ошибок ИИ:** Обсудите, где LLMA может неправильно интерпретировать действия пользователя, если у него нет доступа к рассуждениям (то есть, данным о мыслях).

### 3. **Оценка воздействия:**
   - Оцените, как данные о мыслях могут помочь улучшить работу LLMA.
   - Оцените, без создания модели, как данные о мыслях могут помочь LLMA в разъяснении неоднозначных действий, исправлении недоразумений или лучшей адаптации к интеракциям пользователя.
